# SMS 휴대폰 본인확인(OTP) 설정 가이드

서명 전에 **등록된 의뢰인 연락처로 인증번호(문자)를 보내 본인 확인**을 하는 기능입니다.
아래 순서를 다 마친 뒤 **마지막에 스위치를 켜면** 적용됩니다. (스위치 켜기 전까진 기존 서명 그대로)

> **동작 요약**
> 서명 페이지 → [인증번호 받기] → 등록번호로 문자 발송 → 서명자가 코드 입력 → 확인 →
> 인증 완료돼야 [서명 제출] 활성화. 서버(sign_submit)도 인증을 재확인하므로 우회 불가.

---

## 준비물 체크리스트
- [ ] 솔라피(Solapi) 계정 + API Key/Secret
- [ ] **발신번호 사전등록** (법무법인 번호) — 이게 제일 오래 걸립니다(보통 당일~2일)
- [ ] Supabase에 시크릿 3개 등록
- [ ] Edge Function `otp-send` 배포 (JWT 검증 끄기)
- [ ] OTP SQL 실행 (`supabase/migrations/20260705_sms_otp.sql`)
- [ ] 스위치 켜기 (`OTP_ENABLED=true`) 후 재배포

---

## ① 솔라피 가입 + 발신번호 등록

1. **https://solapi.com** 가입 (사업자번호로).
2. **발신번호 등록**: 콘솔 → 발신번호 관리 → 법무법인 번호(예: 032-868-7171) 등록.
   - 통신서비스 이용증명원 또는 ARS 인증이 필요합니다(전기통신사업법상 필수).
   - **승인돼야 문자가 나갑니다.** 먼저 신청해 두세요.
3. **API 키 발급**: 콘솔 → 개발/연동 → API Key 생성 → **API Key**와 **API Secret**을 복사(시크릿은 이때만 보임).
4. 충전: 소액 충전(문자 1건 ≈ 10~20원).

---

## ② Supabase 시크릿 등록

Supabase 대시보드 → **Edge Functions → Secrets**(또는 Project Settings → Edge Functions) 에서 3개 추가:

| 이름 | 값 |
|---|---|
| `SOLAPI_KEY` | 솔라피 API Key |
| `SOLAPI_SECRET` | 솔라피 API Secret |
| `SMS_SENDER` | 등록한 발신번호(숫자만, 예: `0328687171`) |

> `SUPABASE_URL`·`SUPABASE_SERVICE_ROLE_KEY`는 Supabase가 자동 주입하므로 따로 넣지 않아도 됩니다.

---

## ③ Edge Function 배포 (`otp-send`)

**방법 A — 대시보드(간단):**
1. 대시보드 → **Edge Functions** → **Create a function** → 이름 `otp-send`.
2. 저장소의 **`supabase/functions/otp-send/index.ts`** 내용을 전부 붙여넣고 **Deploy**.
3. 그 함수의 설정에서 **"Verify JWT" 를 끕니다(OFF)**. (서명자는 로그인 안 한 익명이라 필수)

**방법 B — CLI:**
```
supabase functions deploy otp-send --no-verify-jwt
```

---

## ④ OTP SQL 실행

Supabase 대시보드 → **SQL Editor** → 저장소의
**`supabase/migrations/20260705_sms_otp.sql`** 전체 붙여넣고 **Run**.
- `sign_otp` 테이블, 검증 함수, 그리고 **OTP를 요구하는 sign_submit**이 설치됩니다.
- ⚠️ 이 SQL을 실행하면 그때부터 서버가 서명 시 OTP를 요구합니다.
  그러니 **⑤ 스위치 켜기와 같이** 진행하세요(안 그러면 앱에서 인증 UI 없이 막힘).

---

## ⑤ 스위치 켜기 (앱)

`index.html`에서 아래 한 줄을 바꾸고 저장 → 커밋/푸시(자동 배포):
```
var OTP_ENABLED=false;   →   var OTP_ENABLED=true;
```
> 원하시면 이 마지막 단계는 저(클로드)에게 "OTP 켜줘"라고 하시면 바로 바꿔 배포해 드립니다.

---

## ⑥ 테스트
1. 계약서에서 **의뢰인 연락처가 채워진 상태**로 서명요청 링크를 만듭니다(그 번호로 문자가 갑니다).
2. 링크를 열어 → **[인증번호 받기]** → 문자 수신 확인 → 코드 6자리 입력 → **확인** → "본인확인 완료 ✓"
3. 서명 그린 뒤 **[서명 제출]** 이 활성화되는지 확인.
4. 서명 이력(직원 화면)에 **휴대폰 인증** 기록이 남습니다.

---

## 자주 겪는 문제
- **"등록된 연락처가 없어…"**: 서명요청 만들 때 의뢰인 연락처가 비어 있던 경우. 연락처 채워 새 링크 발급.
- **문자가 안 옴**: 발신번호 승인 전이거나, `SMS_SENDER`가 등록번호와 다름/하이픈 포함. 숫자만 넣었는지 확인.
- **"otp_required"로 서명이 막힘**: ④ SQL은 실행했는데 ⑤ 스위치를 안 켠 경우(또는 반대). 둘을 같이 맞추세요.
- **되돌리기**: `OTP_ENABLED=false`로 바꿔 재배포하면 인증 없이 서명(단, ④ SQL의 sign_submit은 여전히 OTP를 요구하므로, 완전 롤백하려면 `20260705_sign_submit_integrity.sql`을 다시 실행해 OTP 없는 버전으로 되돌리세요).

## 참고: 법적 의미
- SMS OTP는 **"등록된 번호를 소지한 사람이 서명했다"**를 증명합니다(휴대폰 소지 확인).
  실명 자체를 검증하는 **통신사 본인확인(PASS 등)**보다는 약하지만, 지금(번호 없이 타이핑)보다
  귀속성이 크게 강화되고, 발송·열람·서명 시각 기록과 결합해 증거력을 높입니다.
- 더 강한 실명확인이 필요하면 본인확인기관(나이스·다날 등) 연동(별도 계약·비용)을 검토하세요.
