# 법무법인 정서 PWA — 개발 가이드

> 이 문서는 앞으로 이 앱을 유지보수하거나 기능을 추가할 때 **꼭 먼저 읽는 규칙서**입니다.
> 코딩이 익숙하지 않아도 이 순서만 지키면 안전하게 작업할 수 있습니다.

---

## 1. 이 프로젝트는 어떻게 생겼나

앱은 여러 파일로 나뉘어 있고, 각자 역할이 정해져 있습니다.

| 파일 | 역할 | 언제 여나요? |
|------|------|-------------|
| `index.html` | 화면 뼈대 + 계약서·선임계·민가사·항소장·서명 로직 | 기존 기능을 고칠 때 |
| `styles.css` | **디자인 전부** (색·폰트·간격·모서리) | 디자인을 바꿀 때 |
| `gyeongyu.js` | 경유(경유증 작성 도우미) 기능 | 경유 기능을 고칠 때 |
| `dojang.html` | 도장 찍기 페이지 (별도 화면) | 도장 기능을 고칠 때 |
| `sw.js` | 서비스워커 (앱 설치·캐시·업데이트) | **배포할 때마다 버전 올림** |
| `manifest.json` | 앱 설치 설정 (이름·아이콘) | 거의 안 건드림 |
| `watermark.png` `firm-stamp.png` | 계약서 워터마크·법인 도장 이미지 | 이미지를 바꿀 때 |

## 2. ⭐ 가장 중요한 규칙 — 새 기능은 무조건 별도 파일

**앞으로 새 기능을 추가할 때는 절대 `index.html` 안에 코드를 쏟아넣지 마세요.**
경유(`gyeongyu.js`)처럼 **새 `.js` 파일**을 만들어서 붙입니다.

### 왜 그런가요?
`index.html`이 이미 큽니다. 여기에 계속 코드를 넣으면 다시 뒤죽박죽이 됩니다.
새 기능을 별도 파일로 두면, 그 기능만 따로 관리할 수 있고 다른 기능이 깨질 위험이 없습니다.

### 새 기능 추가 4단계

**1) 새 파일 만들기** — 예: 새 "위임장" 기능이면 `wijang.js`

```javascript
/* 법무법인 정서 PWA - 위임장 기능
   진입점: "위임장" 버튼 onclick="openWijang()"
   의존성: (다른 기능의 함수를 쓴다면 여기 적어두기) */

function openWijang() {
  // 기능 코드...
}
```

**2) `index.html`에 버튼과 로드 태그 추가**

버튼(원하는 위치에):
```html
<button onclick="openWijang()">위임장 작성</button>
```

파일 로드(`</body>` 바로 앞, 다른 `<script src>`들과 나란히):
```html
<script src="./wijang.js"></script>
```

**3) `sw.js`에 파일 등록 + 버전 올리기** (아래 4번 배포 규칙 참고)

**4) 배포 후 실제로 버튼을 눌러 작동 확인**

## 3. 디자인 바꾸는 법 (색·폰트)

**`styles.css` 맨 위 `:root` 부분의 변수 값만 바꾸면 앱 전체가 바뀝니다.**
개별 화면을 하나하나 고칠 필요가 없습니다.

```css
:root{
  --brand:#1a2740;   /* ← 이 색을 바꾸면 메인 색이 전부 바뀜 */
  --danger:#c0392b;  /* 경고·삭제 (빨강) */
  --success:#1a7f3c; /* 완료 (초록) */
  --info:#1a4ea2;    /* 정보 (파랑) */
  --font: ...;        /* UI 글꼴 */
  --r:12px;           /* 모서리 둥글기 */
}
```

예를 들어 메인 색을 따뜻한 느낌으로 바꾸고 싶으면 `--brand` 한 줄만 수정하면 됩니다.

> ⚠️ **주의:** 계약서·항소장 같은 **법률 문서용 글꼴(함초롬바탕 등)은 `:root` 변수가 아니라 따로 지정**되어 있습니다. 문서 가독성을 위해 일부러 그렇게 둔 것이니 건드리지 마세요.

## 4. ⭐ 배포 체크리스트 (매번 반드시)

`index.html`이나 다른 파일을 바꿔서 GitHub에 올릴 때는 **항상** 아래를 확인하세요.

- [ ] **`sw.js`의 버전을 올렸는가?** — `jeongseo-v58` → `jeongseo-v59`처럼 숫자 +1
- [ ] `index.html`의 `APP_VERSION`도 같은 숫자로 맞췄는가?
- [ ] 새 파일을 추가했다면 **`sw.js`의 `ASSETS` 목록에 그 파일을 넣었는가?**
- [ ] 배포 후 확인:
  - **아이폰 앱**: 홈 화면에서 앱을 지웠다가 다시 설치 (그래야 새 버전 적용)
  - **PC 브라우저**: `Ctrl + Shift + R` (강력 새로고침)

> 이 버전 올리기를 빠뜨리면, 사용자 폰에 **옛날 버전이 계속 남아서** 바뀐 게 안 보입니다. 가장 흔한 실수이니 꼭 챙기세요.

## 5. (참고) 나중에 삼형제를 분리하고 싶어지면

선임계·민가사·항소장 세 기능은 서로 얽혀 있어서 지금은 `index.html`에 함께 두었습니다.
만약 나중에 이 셋을 별도 파일(`legal-docs.js`)로 빼고 싶어지면, `index.html` 안에 이미
**표지판 주석**을 달아두었으니 그걸 따라가면 됩니다.

```
/* ▼▼▼ [선임계 START] ▼▼▼ */   ← 여기부터
   ...
/* ▲▲▲ [선임계 END] ▲▲▲ */     ← 여기까지가 선임계
```

`[민가사 START/END]`, `[항소장 START/END]` 표지판도 있습니다.
단, `⛔ PWA 핵심 코드` 표시가 된 부분은 **절대 함께 옮기면 안 됩니다** (앱 설치·업데이트의 심장부).

## 핵심 요약 (딱 3가지만 기억)

1. **새 기능** → 무조건 새 `.js` 파일로 (index.html에 넣지 않기)
2. **디자인 변경** → `styles.css`의 `:root` 변수만 수정
3. **배포할 때** → `sw.js` 버전 올리기 (안 하면 옛 버전이 남음)
